# Lab20 – SRP: Декомпозиція OrderProcessor

## Тема
Застосування принципу **єдиної відповідальності (SRP)** для декомпозиції складного класу `OrderProcessor` на менші, більш сфокусовані компоненти.

## Мета
- Ознайомитися з принципом SRP та його застосуванням у C#.  
- Навчитися розбивати клас із кількома відповідальностями на окремі класи.  
- Продемонструвати обробку замовлень (`Order`) з використанням SRP.

## Опис реалізації

### Клас Order
Містить дані замовлення:
- `Id` – унікальний ідентифікатор;  
- `CustomerName` – ім’я клієнта;  
- `TotalAmount` – сума замовлення (тип `decimal`);  
- `Status` – статус замовлення (`OrderStatus`);  
- `CreatedAt` – дата та час створення замовлення.
