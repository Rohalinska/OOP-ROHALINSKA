# Звіт з аналізу SOLID принципів (SRP, OCP) в Open-Source проєкті

### Обраний проєкт

Назва: AutoMapper
GitHub: https://github.com/jimmybogard/AutoMapper

Акаунт: jimmybogard
Опис: AutoMapper — open-source бібліотека для автоматичного мапінгу об’єктів у .NET-застосунках.

## Аналіз SRP (Single Responsibility Principle)
### Приклади дотримання SRP

MapperConfiguration
Відповідає лише за зберігання та ініціалізацію конфігурації мапінгу.
Не виконує сам процес мапінгу.

Mapper
Має одну відповідальність — виконання мапінгу об’єктів на основі готової конфігурації.

Profile
Призначений виключно для опису правил мапінгу між типами.
Дає змогу групувати конфігурацію без змішування логіки.

TypeMap
Відповідає лише за зберігання інформації про відповідність між двома типами.

### Порушення SRP

Mapper (частково)
У деяких сценаріях клас Mapper:

виконує мапінг,

працює з контекстом,

взаємодіє з сервісами резолвінгу.

Це створює ознаки перевантаженої відповідальності, хоча зроблено з метою спрощення API.

### Аналіз OCP (Open/Closed Principle)
Приклади дотримання OCP

### IValueResolver
Дозволяє додавати нові стратегії отримання значень
без зміни існуючого коду бібліотеки.
#
public class AgeResolver : IValueResolver<UserEntity, UserDto, int>
{
    public int Resolve(UserEntity source, UserDto destination, int destMember, ResolutionContext context)
    {
        return DateTime.Now.Year - source.BirthDate.Year;
    }
}
#

### IMemberValueResolver
Реалізує патерн Strategy для кастомної логіки мапінгу окремих полів.

### Profile
Нові правила мапінгу додаються через створення нових профілів,
не змінюючи існуючі.
#
public class UserProfile : Profile
{
    public UserProfile()
    {
        CreateMap<UserEntity, UserDto>()
            .ForMember(dest => dest.FullName,
                       opt => opt.MapFrom(src => src.FirstName + " " + src.LastName));
    }
}
#
### ITypeConverter
Дає можливість створювати власні конвертери типів без модифікації ядра AutoMapper.

## Порушення OCP

Умовна логіка для обробки типів
У деяких частинах внутрішньої реалізації використовується if-else
для обробки різних сценаріїв мапінгу, що ускладнює розширення
без змін існуючого коду.

## Висновки

AutoMapper демонструє хороше дотримання SRP, розділяючи конфігурацію, мапінг та резолвінг.

### OCP реалізований через:

інтерфейси,

шаблон Strategy,

можливість розширення через профілі та резолвери.

Деякі компромісні рішення зроблені для зручності використання бібліотеки,
що може знижувати архітектурну чистоту, але покращує UX для розробників.

Проєкт є вдалим прикладом практичного застосування SOLID принципів в open-source середовищі.
