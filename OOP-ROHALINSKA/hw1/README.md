### God Object і принцип SRP
God Object - це коли в програмі є один клас, який робить майже все.
Він створює об’єкти, працює з базою даних, відправляє повідомлення і ще виконує іншу логіку.
Через це клас стає дуже великий і незрозумілий.

Проблема в тому, що якщо потрібно щось змінити, наприклад спосіб збереження даних, то доводиться лізти в цей великий клас.
Через це легко щось зламати, навіть якщо зміна маленька.
Такі класи важко тестувати і підтримувати.

Прикладом God Object може бути клас OrderProcessor.
Він створює замовлення, зберігає його в базі даних і відправляє email користувачу.
Це різні задачі, але вони зібрані в одному класі.

Це порушує принцип SRP.
SRP означає, що клас повинен мати тільки одну відповідальність і одну причину для змін.
Якщо клас робить багато різних речей, то SRP не виконується.
Щоб вирішити цю проблему, клас потрібно розділити на кілька менших.

Окремий клас - для створення замовлення, окремий - для роботи з базою даних, і ще один - для відправки email.
У результаті кожен клас відповідає тільки за свою задачу.
Такий підхід робить код простішим, зрозумілішим і зручнішим для подальшої роботи.

### Приклад класу, який робить занадто багато

public class OrderProcessor

    public void CreateOrder()
    {
        Console.WriteLine("Order created");
    }

    public void SaveToDatabase()
    {
        Console.WriteLine("Order saved");
    }

    public void SendEmail()
    {
        Console.WriteLine("Email sent");
    }
    
Цей клас:

створює замовлення;

зберігає його;

надсилає email.

Це різні задачі, тому принцип SRP порушено.

### Як зробити правильно

Розділяємо задачі на окремі класи

public class OrderService

{

    public void CreateOrder()
    {
        Console.WriteLine("Order created");
    }
}

public class OrderRepository

{
    
    public void Save()
    {
        Console.WriteLine("Order saved");
    }
}

public class EmailService

{

    public void SendEmail()
    {
        Console.WriteLine("Email sent");
    }
}

Тепер:

кожен клас робить одну справу;

код легше читати;

легше змінювати і тестувати.

### Висновок

God Object - це коли один клас робить все підряд.
SRP каже: один клас - одна відповідальність.
Так код стає простішим і зрозумілішим.
